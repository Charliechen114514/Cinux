# 030-1 PS/2 鼠标驱动与 GUI 事件系统

## 导语

在上一个 tag 里我们把 Canvas 双缓冲画布和 PSF2 字体渲染跑通了，屏幕上已经能画矩形、写字了。但一个真正能用的 GUI 不能只有输出——我们还需要输入。这一章我们要做的就是把 PS/2 鼠标驱动从零写出来，同时搭好 GUI 子系统的事件队列，把键盘和鼠标的输入统一到一个地方，让后面的窗口管理器可以从一个入口消费所有用户操作。

完成本章后，你会看到一个能响应鼠标移动和点击的内核，串口输出会打印鼠标坐标变化和按键状态，而且键盘事件也会同时被推送到 GUI 事件队列里。

知识前置：你需要了解 PS/2 键盘驱动的基本工作方式（端口 0x60/0x64、IRQ1 中断处理），这些在 tag 014 已经讲过了。

## 概念精讲

### 8042 PS/2 控制器的辅助端口

你可能还记得键盘驱动里我们跟 8042 控制器打交道的那些端口：0x60 是数据口，0x64 是命令/状态口。实际上 8042 是一个双通道控制器——第一通道接键盘（IRQ1），第二通道就是所谓的"辅助端口"（auxiliary port），也就是鼠标口（IRQ12）。两者的通信方式几乎一模一样，只是鼠标需要先通过控制器命令 0xD4 告诉 8042"下一条指令是给鼠标的"。

你可以把它理解成一个有两个口的邮局：键盘和鼠标各有一个信箱，但对外只有一个地址。要给鼠标寄信，你得先说"这封给二号信箱"，然后再塞内容。

### PS/2 鼠标的 3 字节包格式

PS/2 鼠标在 streaming 模式下每次移动或按键都会发一个 3 字节的包。第一个字节编码了按键状态和方向符号位——bit 0 是左键，bit 1 是右键，bit 2 是中键，bit 3 永远是 1（这是包同步标记），bit 4 和 bit 5 分别是 X 和 Y 的符号位，bit 6 和 bit 7 是溢出标志。第二字节是 X 方向的相对位移，第三字节是 Y 方向的相对位移。这里有一个非常重要的细节：位移值是 9 位有符号数，低 8 位在第二/三字节，第 9 位（符号位）在第一字节的 bit 4/5 里。如果你直接把第二字节当 8 位有符号数用，快速移动鼠标的时候就会出现诡异的行为，因为负数的补码表示会被截断。

Y 轴方向还有一个容易搞反的点：PS/2 协议里正 dy 表示鼠标物理上向上移动，但屏幕坐标系 Y 轴是向下的，所以累积的时候要减去 dy 而不是加上去。

### 环形缓冲区事件队列

有了鼠标和键盘两种输入源之后，我们需要一个统一的地方来收集所有事件。这里用的是经典的环形缓冲区（ring buffer）：一个固定大小的数组，一个 head 指针（读端），一个 tail 指针（写端）。中断处理函数（生产者）往 tail 端写，窗口管理器（消费者）从 head 端读。队列满的时候直接丢弃新事件——在内核 GUI 里这比阻塞要安全得多，毕竟你不能在 IRQ handler 里等消费者腾空间。

我们的队列容量是 128 个事件，每个事件是一个带类型标签的联合体，可以是鼠标事件也可以是键盘事件。消费者拿到事件后看类型标签就知道该怎么处理。

## 动手实现

### Step 1: 鼠标驱动的 8042 初始化

**目标**: 让 8042 控制器开启辅助端口并配置 IRQ12 中断。

**设计思路**: 初始化分三步走。第一步，向命令口 0x64 发送 0xA8 启用辅助端口（鼠标口）。第二步，读取控制器配置字节（0x20），把 bit 1 设成 1 以启用 IRQ12，然后写回（0x60）。第三步，通过 0xD4 前缀命令告诉控制器"接下来发给鼠标"，然后发送 0xF4 开启鼠标的 streaming 模式。鼠标应该回复 0xFA（ACK）。注意每一步操作之前都要等待输入缓冲区空（读 0x64 的 bit 1），读取响应之前要等待输出缓冲区满（读 0x64 的 bit 0），超时保护也要加上。

**踩坑预警**: 初始化过程中操作 PS/2 控制器可能触发一个虚假的 IRQ1（键盘中断）。这是 8042 硬件的已知行为，不是你的 bug。前提是你的 ISR 栈对齐是正确的——下一章会专门讲这个。

**验证**: 在 init 函数末尾通过串口打印初始化完成信息和 ACK 状态。启动后用 `make run` 观察 `[MOUSE] Mouse enabled (ACK received).` 和 `[MOUSE] PS/2 mouse driver initialised.` 两行输出。

### Step 2: IRQ12 中断处理与 3 字节包解析

**目标**: 在 IRQ12 中断里逐字节积累鼠标包，解析出位移和按键状态。

**设计思路**: 鼠标每次中断送来一个字节，所以你需要维护一个 3 字节的缓冲区和一个索引计数器。每来一个字节就存进去、索引加一，满 3 个就解析。这里有一个关键的同步机制：第一个字节（byte 0）的 bit 3 永远是 1，如果索引是 0 但当前字节 bit 3 不是 1，说明我们丢同步了，直接丢弃等下一个包开头。

解析的时候要正确提取 9 位有符号位移：把第二字节转成 32 位整数，如果第一字节的符号位（bit 4 对 X，bit 5 对 Y）是 1，就减去 256 来做符号扩展。Y 轴记得取反。

**实现约束**: 驱动类设计成全静态方法——系统里只有一个 PS/2 鼠标，不需要实例化。静态成员存储包缓冲区、当前光标坐标、屏幕尺寸边界、按键状态（当前和上一帧，用于边缘检测）。

**验证**: 在 decode_packet 里临时加一行串口打印，每次解析完打印当前坐标和按键。移动鼠标应该能看到坐标变化，按下左键应该看到按钮状态变化。

### Step 3: 绝对坐标累积与屏幕边界裁剪

**目标**: 从 PS/2 的相对位移推算出屏幕上的绝对光标位置。

**设计思路**: 维护两个 int32_t 的静态变量 mouse_x 和 mouse_y，初始值设为 0。每次解析出 dx/dy 后，mouse_x += dx, mouse_y -= dy（Y 轴取反）。然后裁剪到 [0, screen_width - 1] 和 [0, screen_height - 1]。屏幕尺寸通过一个 set_screen_bounds 函数设置，默认值 1024x768。

**踩坑预警**: 别忘了在 set_screen_bounds 里把当前坐标也裁剪一下新边界——如果屏幕分辨率中途改了，光标可能已经越界。

**验证**: 设置屏幕边界为你的 QEMU 分辨率，移动鼠标到四个角，确认坐标不会超过边界值。

### Step 4: 事件队列与双路分发

**目标**: 鼠标驱动把解析好的事件推入 GUI 事件队列；键盘驱动同时推入自己的队列和 GUI 队列。

**设计思路**: 事件队列是一个环形缓冲区模板，容量 128。事件结构体包含一个枚举类型标签和鼠标/键盘数据的联合体。鼠标事件类型分 MouseMove（有位移时触发）、MouseDown（新按下的按钮）、MouseUp（新释放的按钮）。按键边缘检测通过对比当前和上一次的按钮位掩码实现。

键盘驱动的双路分发在 irq1_handler 里用条件编译（CINUX_GUI 宏）控制：解析完 scan code 之后，除了推入键盘自己的队列，还构造一个 GUI Event 推入鼠标驱动持有的全局事件队列。这样窗口管理器只需要从一个队列就能读到所有输入。

**实现约束**: enqueue 在队列满的时候静默丢弃事件。dequeue 在队列空的时候返回 false。这两个操作都不需要锁——中断上下文是唯一的生产者，窗口管理器在正常上下文消费时需要关中断保护。

**验证**: 在串口日志中统计收到的事件数量。移动鼠标+按键应该看到 MouseMove、MouseDown、MouseUp 事件。敲键盘应该看到 KeyDown/KeyUp 事件出现在同一个队列里。

## 构建与运行

完整的构建和运行流程跟之前一样。QEMU 启动参数需要确保鼠标设备可用（QEMU 默认就有 PS/2 鼠标）。构建命令：

```
make run
```

如果遇到鼠标在 VNC 里不响应的问题，可以尝试添加 USB tablet 设备参数。不过 PS/2 鼠标在 QEMU 里是默认开启的，通常不需要额外配置。

## 调试技巧

最常见的 bug 是包解析丢同步——如果你的光标坐标突然跳到很奇怪的值，检查 byte 0 的 bit 3 同步逻辑有没有生效。你可以在 process_byte 里加串口打印，记录每个收到的字节和当前索引，很容易就能看出是不是有字节漏了。

第二个常见问题是 IRQ12 没有正确路由到 IDT。确认你的 interrupts.S 里 irq12_stub 对应的向量号是 0x2C（IRQ12 + 0x20），并且 irq_init 里确实注册了这个向量。

第三个坑是忘了在 irq12_handler 末尾发送 EOI。如果忘了，后续的中断就不会再来，鼠标会"冻住"。PIC::send_eoi(12) 不可少——注意 12 号 IRQ 在从片上，所以 send_eoi 需要同时通知从片和主片。

## 本章小结

| 概念 | 关键点 |
|------|--------|
| 8042 辅助端口 | 命令 0xA8 启用, 0xD4 转发到鼠标 |
| 3 字节包 | byte0 = 按键+符号+同步, byte1 = dx, byte2 = dy |
| 9 位有符号 delta | 低 8 位在 byte1/2, 符号位在 byte0 bit4/5 |
| Y 轴方向 | PS/2 正 dy = 物理向上 = 屏幕 Y 减 |
| 环形缓冲区 | 固定容量, 满则丢, 无锁单生产者单消费者 |
| 双路分发 | 键盘 IRQ1 同时推入键盘队列和 GUI 队列 |
| IRQ12 路由 | 向量 0x2C, IDT 注册, EOI 通知从片 |
