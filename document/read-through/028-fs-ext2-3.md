# 028-3 Read-through: inode 操作与文件读写

## 概览

前两篇我们看完了超级块、块组描述符这些"搭台子"的工作，现在进入驱动最核心的数据路径——inode 操作与文件读写。我们要拆解的代码覆盖 inode 缓存管理（`get_cached_inode`、`populate_vfs_inode`）、磁盘 inode 持久化（`write_disk_inode`）、块指针解析与分配（`get_or_alloc_block`），以及文件级读写回调（`Ext2FileOps::read`/`write`）。这些函数构成了一条从 VFS `Inode` 出发、经过缓存查找、块寻址、最终落到 AHCI 磁盘 I/O 的完整通路。

## 架构图

```
读取链路:
  inode->ops->read → Ext2FileOps::read
    ├── fs_private → Ext2CachedInode → disk_inode
    ├── file_block < 12 → i_block[file_block]
    ├── file_block < 12+ptrs → 间接块查表
    ├── disk_block == 0 → 空洞填零
    └── read_block → memcpy

写入链路:
  inode->ops->write → Ext2FileOps::write
    ├── get_or_alloc_block → 按需分配
    ├── read-modify-write → 打补丁
    ├── 更新 i_size / i_blocks
    └── write_disk_inode → 持久化
```

## 代码精讲

### Ext2FileOps 与 Ext2DirOps —— C++ 虚基类的 InodeOps 实现

```cpp
class Ext2FileOps : public InodeOps {
public:
    explicit Ext2FileOps(Ext2& ext2);
    int64_t read(const Inode* inode, uint64_t offset, void* buf, uint64_t count) override;
    int64_t write(Inode* inode, uint64_t offset, const void* buf, uint64_t count) override;
    int64_t stat(const Inode* inode, struct stat* st) override;
private:
    Ext2& ext2_;
};
```

Cinux 的 VFS 定义了 `InodeOps` 虚基类，ext2 用两个子类分别处理文件和目录操作。`Ext2FileOps` 持有 `Ext2&` 引用——相比上一版的全局指针 `g_ext2_instance`，干净得多。构造函数就是一个初始化列表：`Ext2FileOps::Ext2FileOps(Ext2& ext2) : ext2_(ext2) {}`。每个 VFS `Inode` 的 `ops` 在 `populate_vfs_inode` 中被设为 `&file_ops_` 或 `&dir_ops_`，分发走虚函数调用，零全局状态。

### inode 缓存：get_cached_inode

```cpp
Inode* Ext2::get_cached_inode(uint32_t ino) {
    // 第一阶段：线性扫描，命中直接返回
    for (uint32_t i = 0; i < EXT2_INODE_CACHE_SIZE; ++i) {
        if (inode_cache_[i].in_use && inode_cache_[i].ino == ino)
            return &inode_cache_[i].vfs_inode;
    }
    // 第二阶段：找空闲槽位，从磁盘读取并填充
    for (uint32_t i = 0; i < EXT2_INODE_CACHE_SIZE; ++i) {
        if (!inode_cache_[i].in_use) {
            if (!read_disk_inode(ino, inode_cache_[i].disk_inode)) return nullptr;
            inode_cache_[i].ino = ino;
            inode_cache_[i].in_use = true;
            populate_vfs_inode(inode_cache_[i]);
            return &inode_cache_[i].vfs_inode;
        }
    }
    // 第三阶段：缓存满了，伪随机驱逐
    uint32_t evict = 1 + (ino % (EXT2_INODE_CACHE_SIZE - 1));
    inode_cache_[evict].in_use = false;
    if (!read_disk_inode(ino, inode_cache_[evict].disk_inode)) return nullptr;
    inode_cache_[evict].ino = ino;
    inode_cache_[evict].in_use = true;
    populate_vfs_inode(inode_cache_[evict]);
    return &inode_cache_[evict].vfs_inode;
}
```

固定 64 槽位的 `Ext2CachedInode` 数组，三阶段查找：线性扫描命中、空闲槽位冷加载、都失败就驱逐。驱逐公式 `1 + (ino % 63)` 保证结果落在 1-63，永远踢不到 slot 0——那是根目录的家。

### populate_vfs_inode —— 磁盘 inode 到 VFS Inode 的翻译

```cpp
void Ext2::populate_vfs_inode(Ext2CachedInode& cached) {
    const Ext2Inode& disk = cached.disk_inode;
    cached.vfs_inode.ino  = cached.ino;
    cached.vfs_inode.size = disk.i_size;

    uint16_t mode_type = disk.i_mode & EXT2_S_IFMT;
    if (mode_type == EXT2_S_IFDIR) {
        cached.vfs_inode.type = InodeType::Directory;
        cached.vfs_inode.ops  = &dir_ops_;
    } else if (mode_type == EXT2_S_IFREG) {
        cached.vfs_inode.type = InodeType::Regular;
        cached.vfs_inode.ops  = &file_ops_;
    } else {
        cached.vfs_inode.type = InodeType::Unknown;
        cached.vfs_inode.ops  = nullptr;
    }
    cached.vfs_inode.fs_private = &cached;
    cached.vfs_inode.mode   = disk.i_mode;
    cached.vfs_inode.nlink  = disk.i_links_count;
    cached.vfs_inode.blocks = disk.i_blocks;
    // ... 以及 uid/gid/atime/ctime/mtime
}
```

这个函数把 128 字节的 `Ext2Inode` 翻译成 VFS 的 `Inode`。关键是根据 `i_mode` 类型位设置 `ops` 指针：目录走 `dir_ops_`，文件走 `file_ops_`。`fs_private = &cached` 这个回指指针让回调可以直接拿到完整磁盘 inode，零额外查找。

### write_disk_inode —— read-modify-write 持久化

```cpp
bool Ext2::write_disk_inode(uint32_t ino, const Ext2Inode& inode) {
    if (ino == 0) return false;
    uint32_t group = (ino - 1) / inodes_per_group_;
    if (group >= group_count_) return false;

    // 定位：块组 → inode 表 → 块内偏移
    uint32_t inode_table_block   = bgdt_[group].bg_inode_table;
    uint32_t index_in_group      = (ino - 1) % inodes_per_group_;
    uint64_t byte_offset         = static_cast<uint64_t>(index_in_group) * inode_size_;
    uint32_t block_offset        = static_cast<uint32_t>(byte_offset / block_size_);
    uint32_t within_block_offset = static_cast<uint32_t>(byte_offset % block_size_);
    uint32_t target_block        = inode_table_block + block_offset;

    // read-modify-write 三步走
    if (!read_block(target_block)) return false;
    if (within_block_offset + sizeof(Ext2Inode) > block_size_) return false;
    auto* block_data = reinterpret_cast<uint8_t*>(dma_buf_virt_);
    memcpy(block_data + within_block_offset, &inode, sizeof(Ext2Inode));
    if (!write_block(target_block)) return false;
    return true;
}
```

read-modify-write：一个 4KB 块里塞了多个 inode（128 字节各一个），磁盘 I/O 最小单位是整块，所以先读、覆盖目标 inode、再写回。定位过程和 `read_disk_inode` 完全对称——块组 → 组内索引 → 字节偏移 → 磁盘块号 + 块内偏移。

### get_or_alloc_block —— 块指针解析器（含分配）

```cpp
uint32_t Ext2::get_or_alloc_block(Ext2Inode& disk, uint32_t file_block) {
    // 直接块 0-11：指针为零则分配
    if (file_block < EXT2_DIRECT_BLOCKS) {
        if (disk.i_block[file_block] == 0) {
            uint32_t blk = alloc_block();
            if (blk == 0) return 0;
            auto* dma = reinterpret_cast<uint8_t*>(dma_buf_virt_);
            for (uint32_t i = 0; i < block_size_; ++i) dma[i] = 0;
            if (!write_block(blk)) { free_block(blk); return 0; }
            disk.i_block[file_block] = blk;
        }
        return disk.i_block[file_block];
    }
    // 间接块 12+：按需分配间接块本身，再按需分配数据块
    if (file_block < EXT2_DIRECT_BLOCKS + block_size_ / sizeof(uint32_t)) {
        uint32_t indirect_idx = file_block - EXT2_DIRECT_BLOCKS;
        if (disk.i_block[EXT2_INDIRECT_BLOCK] == 0) {
            uint32_t indirect_blk = alloc_block();
            if (indirect_blk == 0) return 0;
            auto* dma = reinterpret_cast<uint8_t*>(dma_buf_virt_);
            for (uint32_t i = 0; i < block_size_; ++i) dma[i] = 0;
            if (!write_block(indirect_blk)) { free_block(indirect_blk); return 0; }
            disk.i_block[EXT2_INDIRECT_BLOCK] = indirect_blk;
        }
        uint32_t indirect_blk = disk.i_block[EXT2_INDIRECT_BLOCK];
        if (!read_block(indirect_blk)) return 0;
        auto* indirect = reinterpret_cast<uint32_t*>(dma_buf_virt_);
        if (indirect[indirect_idx] == 0) {
            uint32_t data_blk = alloc_block();
            if (data_blk == 0) return 0;
            auto* dma = reinterpret_cast<uint8_t*>(dma_buf_virt_);
            for (uint32_t i = 0; i < block_size_; ++i) dma[i] = 0;
            if (!write_block(data_blk)) { free_block(data_blk); return 0; }
            // DMA 已被覆盖，必须重读间接块再修改
            if (!read_block(indirect_blk)) return 0;
            indirect = reinterpret_cast<uint32_t*>(dma_buf_virt_);
            indirect[indirect_idx] = data_blk;
            if (!write_block(indirect_blk)) return 0;
        }
        if (!read_block(indirect_blk)) return 0;
        return reinterpret_cast<uint32_t*>(dma_buf_virt())[indirect_idx];
    }
    return 0;  // 双重/三重间接暂不支持
}
```

语义是"给我 file_block 对应的磁盘块号，没有就分配一个"。直接块（0-11）为零就分配、清零、写回、填入 `i_block`。间接块更复杂——先确保 `i_block[12]` 存在，再在间接块里查目标槽位，为零就再分配数据块。注意分配数据块后 DMA 缓冲区已被覆盖，必须重读间接块才能安全修改指针。每次分配失败都调用 `free_block` 回收，避免泄漏。

### Ext2FileOps::read —— 分块读取

```cpp
int64_t Ext2FileOps::read(const Inode* inode, uint64_t offset,
                          void* buf, uint64_t count) {
    auto* cached = static_cast<const Ext2CachedInode*>(inode->fs_private);
    const Ext2Inode& disk = cached->disk_inode;
    if (offset >= disk.i_size) return 0;
    uint64_t available = disk.i_size - offset;
    uint64_t to_read   = (count < available) ? count : available;
    uint32_t bs = ext2_.block_size();
    auto* dst = static_cast<uint8_t*>(buf);
    uint64_t total_read = 0;

    while (total_read < to_read) {
        uint64_t file_block   = (offset + total_read) / bs;
        uint64_t block_offset = (offset + total_read) % bs;
        uint64_t chunk = bs - block_offset;
        if (chunk > to_read - total_read) chunk = to_read - total_read;

        uint32_t disk_block = 0;
        if (file_block < EXT2_DIRECT_BLOCKS)
            disk_block = disk.i_block[file_block];
        else if (file_block < EXT2_DIRECT_BLOCKS + bs / sizeof(uint32_t)) {
            uint32_t ib = disk.i_block[EXT2_INDIRECT_BLOCK];
            if (ib == 0 || !ext2_.read_block(ib)) break;
            disk_block = reinterpret_cast<uint32_t*>(
                ext2_.dma_buf_virt())[file_block - EXT2_DIRECT_BLOCKS];
        } else break;

        if (disk_block == 0) {  // 文件空洞
            for (uint64_t i = 0; i < chunk; ++i) dst[total_read + i] = 0;
            total_read += chunk; continue;
        }
        if (!ext2_.read_block(disk_block)) break;
        memcpy(dst + total_read,
               reinterpret_cast<const uint8_t*>(ext2_.dma_buf_virt()) + block_offset,
               chunk);
        total_read += chunk;
    }
    return static_cast<int64_t>(total_read);
}
```

块寻址是 read 的核心。0-11 直接取 `i_block`，12+ 读间接块查表。间接块读取会覆盖 DMA 缓冲区，但 `disk_block` 已提前提取，不影响后续操作。`disk_block == 0` 代表文件空洞，POSIX 语义要求返回全零。`offset >= disk.i_size` 的判断保证不越过文件末尾。

### Ext2FileOps::write —— 分块写入

```cpp
int64_t Ext2FileOps::write(Inode* inode, uint64_t offset,
                           const void* buf, uint64_t count) {
    auto* cached = static_cast<Ext2CachedInode*>(inode->fs_private);
    Ext2Inode& disk = cached->disk_inode;
    uint32_t bs = ext2_.block_size();
    auto* src = static_cast<const uint8_t*>(buf);
    uint64_t total_written = 0;

    while (total_written < count) {
        uint64_t file_block   = (offset + total_written) / bs;
        uint64_t block_offset = (offset + total_written) % bs;
        uint64_t chunk = bs - block_offset;
        if (chunk > count - total_written) chunk = count - total_written;
        if (file_block > EXT2_DIRECT_BLOCKS) break;

        uint32_t disk_block = ext2_.get_or_alloc_block(disk, file_block);
        if (disk_block == 0) break;
        if (block_offset != 0 || chunk != bs) {
            if (!ext2_.read_block(disk_block)) break;      // read-modify-write
        } else {
            auto* dma = reinterpret_cast<uint8_t*>(ext2_.dma_buf_virt());
            for (uint32_t i = 0; i < bs; ++i) dma[i] = 0;  // 整块覆盖直接清零
        }
        auto* dst = reinterpret_cast<uint8_t*>(ext2_.dma_buf_virt()) + block_offset;
        for (uint64_t i = 0; i < chunk; ++i) dst[i] = src[total_written + i];
        if (!ext2_.write_block(disk_block)) break;
        total_written += chunk;
    }
    if (total_written > 0) {
        uint64_t new_end = offset + total_written;
        if (new_end > disk.i_size) {
            disk.i_size = static_cast<uint32_t>(new_end);
            disk.i_blocks = ((disk.i_size + bs - 1) / bs) * (bs / 512);
        }
        ext2_.write_disk_inode(static_cast<uint32_t>(inode->ino), disk);
        inode->size = disk.i_size;
    }
    return static_cast<int64_t>(total_written);
}
```

write 和 read 结构对称，但多了两步。一是 `get_or_alloc_block`：目标块不存在时必须分配新块，不能像 read 那样填零跳过。二是 read-modify-write：非块对齐写入必须先读原始块、打补丁再写回。循环结束后更新 `i_size`/`i_blocks`，通过 `write_disk_inode` 持久化。

`Ext2FileOps::stat` 和 `Ext2DirOps::stat` 实现一致：从 `fs_private` 取 `Ext2CachedInode`，把 `i_mode`、`i_size`、`i_links_count`、时间戳等字段搬到 POSIX `struct stat`，`st_blksize` 设为 `ext2_.block_size()`。两个子类的重复代码可以后续提取到公共辅助函数。

## 设计决策

### 虚基类 InodeOps + 引用注入

上一版用全局指针 `g_ext2_instance`，单实例能用但无法扩展到多磁盘挂载。现在 InodeOps 是虚基类，`Ext2FileOps`/`Ext2DirOps` 持有 `Ext2&` 引用，每个 ops 实例天然知道自己的 Ext2 对象，支持多分区挂载无需任何修改。

### get_or_alloc_block 的懒惰分配

写入时目标块可能不存在。`get_or_alloc_block` 在指针为零时才分配，间接块也按需创建。每次分配后立即清零写回，避免未初始化数据泄露。这和 ext2 原生的文件空洞语义一致——预先分配不仅浪费空间，还破坏稀疏文件支持。

## 扩展方向

- 支持双重间接块（`i_block[13]`），将最大单文件尺寸提升到约 64MB（高）
- inode 缓存升级为 LRU 驱逐，减少热 inode 被误踢（中）
- 写入路径加入 write-ahead log，确保崩溃后元数据一致性（极高）
- 提取 stat 公共逻辑，消除 Ext2FileOps 和 Ext2DirOps 的重复代码（低）
