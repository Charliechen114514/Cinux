# 028-4 Read-through: 目录操作与 VFS 集成

## 概览

本文拆解 ext2 驱动的目录操作、路径解析和文件系统变动操作（create、mkdir、unlink），以及底层的块/inode 分配器和元数据写回机制。这些功能让 ext2 从一个只能读取文件的骨架进化为支持完整文件系统操作的驱动。我们会看到 `Ext2DirOps` 的回调如何委托给 Ext2 核心方法、目录项的 rec_len 切分与合并技巧、以及位图扫描分配器怎样从磁盘上找到空闲资源。

## 架构图

```
目录操作调用链:
  VFS sys_mkdir("/tmp/dir") → Ext2DirOps::mkdir
    └── Ext2::mkdir(parent_ino, name, name_len)
          ├── lookup_in_dir (检查重名)
          ├── alloc_inode (inode 位图扫描)
          ├── alloc_block (block 位图扫描, 给新目录分配数据块)
          ├── 初始化 "."/".." 目录项 → write_block
          ├── add_dir_entry (在父目录插入新条目)
          ├── write_bgdt / write_superblock (更新空闲计数)
          └── get_cached_inode (缓存新 inode 并返回)

块分配器:
  alloc_block() → 遍历块组 → 读块位图 → 找空闲 bit → 置位 → 写回
  alloc_inode() → 遍历块组 → 读 inode 位图 → 找空闲 bit → 置位 → 写回
  free_block()  → 计算块组 → 读位图 → 清 bit → 写回 → 更新计数
  free_inode()  → 计算块组 → 读位图 → 清 bit → 写回 → 更新计数

目录项操作:
  add_dir_entry    → 扫描现有条目 → rec_len 切分 → 或分配新块
  remove_dir_entry → 找目标条目 → rec_len 合并到前驱 → 写回
```

## 代码精讲

### Ext2DirOps::readdir —— 基于索引的目录枚举

```cpp
int64_t Ext2DirOps::readdir(const Inode* inode, uint64_t index, char* name, uint64_t name_max) {
    if (inode == nullptr || inode->fs_private == nullptr || name == nullptr || name_max == 0) {
        return -1;
    }

    auto*            cached = static_cast<const Ext2CachedInode*>(inode->fs_private);
    const Ext2Inode& disk   = cached->disk_inode;

    uint32_t bs = ext2_.block_size();

    if (index == 0) {
        if (name_max < 2) {
            return -1;
        }
        name[0] = '.';
        name[1] = '\0';
        return 1;
    }
    if (index == 1) {
        if (name_max < 3) {
            return -1;
        }
        name[0] = '.';
        name[1] = '.';
        name[2] = '\0';
        return 1;
    }

    uint64_t target   = index - 2;
    uint64_t found    = 0;
    uint64_t dir_size = disk.i_size;

    uint32_t total_blocks = static_cast<uint32_t>((dir_size + bs - 1) / bs);
    if (total_blocks > EXT2_DIRECT_BLOCKS) {
        total_blocks = EXT2_DIRECT_BLOCKS;
    }

    for (uint32_t b = 0; b < total_blocks; ++b) {
        uint32_t blk = disk.i_block[b];
        if (blk == 0) {
            continue;
        }

        if (!ext2_.read_block(blk)) {
            return -1;
        }

        auto*    block_data = reinterpret_cast<const uint8_t*>(ext2_.dma_buf_virt());
        uint32_t pos        = 0;

        while (pos < bs) {
            if (pos + EXT2_DIR_ENTRY_HDR_SIZE > bs) {
                break;
            }

            auto* entry = reinterpret_cast<const Ext2DirEntry*>(block_data + pos);

            if (entry->rec_len == 0) {
                break;
            }

            if (entry->inode != 0) {
                // Skip "." and ".." — they are handled by the
                // hardcoded indices 0 and 1 above
                if (entry->name_len == 1 && entry->name[0] == '.') {
                    pos += entry->rec_len;
                    continue;
                }
                if (entry->name_len == 2 && entry->name[0] == '.' && entry->name[1] == '.') {
                    pos += entry->rec_len;
                    continue;
                }

                if (found == target) {
                    uint32_t copy_len = static_cast<uint32_t>(name_max) - 1;
                    if (entry->name_len < copy_len) {
                        copy_len = entry->name_len;
                    }

                    for (uint32_t i = 0; i < copy_len; ++i) {
                        name[i] = entry->name[i];
                    }
                    name[copy_len] = '\0';
                    return 1;
                }
                ++found;
            }

            pos += entry->rec_len;
        }
    }

    return 0;
}
```

readdir 用索引号而非字节偏移来定位条目。index 0 和 1 硬编码返回 "." 和 ".."——POSIX 要求每个目录都有这两个条目，但不需要每次都从磁盘读。从 index 2 开始才真正扫描磁盘：`target = index - 2` 表示要找的磁盘上第 N 个非 "."/".." 条目。内层循环按 `rec_len` 步进，遇到 "." 和 ".." 条目时跳过它们（否则用户会看到重复），遇到有效条目则递增计数器 `found`，匹配 `target` 时拷贝名称返回 1。返回 0 表示索引越界，调用者知道枚举结束。

### Ext2DirOps 的 create / mkdir / unlink —— 委托给 Ext2 核心

三个方法都是纯粹的委托——参数校验后把 VFS Inode 的 `ino` 提取出来，转发给 Ext2 对应方法。这样 InodeOps 层只关心 VFS 接口适配，所有 ext2 特定的逻辑留在 Ext2 类内部。

### lookup_in_dir / lookup —— 路径解析

```cpp
uint32_t Ext2::lookup_in_dir(uint32_t dir_ino, const char* name, uint32_t name_len) {
    Ext2Inode dir_disk;
    if (!read_disk_inode(dir_ino, dir_disk)) {
        return 0;
    }

    uint32_t bs           = block_size_;
    uint32_t dir_size     = dir_disk.i_size;
    uint32_t total_blocks = (dir_size + bs - 1) / bs;

    if (total_blocks > EXT2_DIRECT_BLOCKS) {
        total_blocks = EXT2_DIRECT_BLOCKS;
    }

    for (uint32_t b = 0; b < total_blocks; ++b) {
        uint32_t blk = dir_disk.i_block[b];
        if (blk == 0) {
            continue;
        }

        if (!read_block(blk)) {
            return 0;
        }

        auto*    block_data = reinterpret_cast<const uint8_t*>(dma_buf_virt_);
        uint32_t pos        = 0;

        while (pos < bs) {
            if (pos + EXT2_DIR_ENTRY_HDR_SIZE > bs) {
                break;
            }

            auto* entry = reinterpret_cast<const Ext2DirEntry*>(block_data + pos);

            if (entry->rec_len == 0) {
                break;
            }

            if (entry->inode != 0 && entry->name_len == name_len) {
                bool match = true;
                for (uint32_t i = 0; i < name_len; ++i) {
                    if (entry->name[i] != name[i]) {
                        match = false;
                        break;
                    }
                }

                if (match) {
                    return entry->inode;
                }
            }

            pos += entry->rec_len;
        }
    }

    return 0;
}
```

`lookup_in_dir` 是路径解析的原子操作：给定一个目录 inode 号和一个组件名称，扫描目录数据块找匹配条目。先比较 `name_len`——长度不同直接跳过，省去逐字节比较。ext2 文件名不以 null 结尾，长度信息存在 `name_len` 字段中，所以这里的逐字节比较是必要的。

```cpp
Inode* Ext2::lookup(const char* path) {
    if (path == nullptr) {
        return nullptr;
    }

    if (path[0] == '\0' || (path[0] == '/' && path[1] == '\0')) {
        return &root_inode_;
    }

    if (path[0] == '/') {
        ++path;
    }

    uint32_t current_ino = 2;

    while (path[0] != '\0') {
        uint32_t comp_len = 0;
        while (path[comp_len] != '\0' && path[comp_len] != '/') {
            ++comp_len;
        }

        if (comp_len == 0) {
            ++path;
            continue;
        }

        uint32_t found_ino = lookup_in_dir(current_ino, path, comp_len);
        if (found_ino == 0) {
            return nullptr;
        }

        path += comp_len;
        if (path[0] == '/') {
            ++path;
        }

        if (path[0] != '\0') {
            Ext2Inode check;
            if (!read_disk_inode(found_ino, check)) {
                return nullptr;
            }
            if ((check.i_mode & EXT2_S_IFMT) != EXT2_S_IFDIR) {
                return nullptr;
            }
        }

        current_ino = found_ino;
    }

    return get_cached_inode(current_ino);
}
```

`lookup` 实现经典的 Unix 路径解析：从根 inode (2) 开始，逐级分解路径组件。每找到一级后，如果后面还有更多组件，就读取 inode 的 `i_mode` 验证是目录——这是防止把普通文件当目录用的防线。最终结果通过 `get_cached_inode` 放入缓存并返回。

### create() —— 创建普通文件

```cpp
Inode* Ext2::create(uint32_t parent_ino, const char* name, uint32_t name_len) {
    if (name == nullptr || name_len == 0 || name_len > EXT2_NAME_MAX) {
        return nullptr;
    }

    if (lookup_in_dir(parent_ino, name, name_len) != 0) {
        return nullptr;
    }

    Ext2Inode dir_disk;
    if (!read_disk_inode(parent_ino, dir_disk)) {
        return nullptr;
    }

    uint32_t new_ino = alloc_inode();
    if (new_ino == 0) {
        return nullptr;
    }

    Ext2Inode new_disk;
    for (uint32_t i = 0; i < sizeof(Ext2Inode); ++i) {
        reinterpret_cast<uint8_t*>(&new_disk)[i] = 0;
    }

    new_disk.i_mode        = EXT2_S_IFREG | 0644;
    new_disk.i_links_count = 1;

    if (!write_disk_inode(new_ino, new_disk)) {
        free_inode(new_ino);
        return nullptr;
    }

    if (!add_dir_entry(parent_ino, dir_disk, new_ino, name, name_len, Ext2FileType::Regular)) {
        free_inode(new_ino);
        return nullptr;
    }

    if (!write_disk_inode(parent_ino, dir_disk)) {
        return nullptr;
    }

    return get_cached_inode(new_ino);
}
```

创建文件分五步：先检查重名，然后分配 inode，初始化为普通文件（mode = REG | 0644，links = 1），写入磁盘，再在父目录中添加目录项。任何步骤失败都会回滚——`free_inode` 回收已分配的 inode 号。`add_dir_entry` 可能修改 `dir_disk`（如果分配了新块），所以最后要写回父目录 inode。

### mkdir() —— 创建目录

```cpp
Inode* Ext2::mkdir(uint32_t parent_ino, const char* name, uint32_t name_len) {
    // ... 参数校验和重名检查 ...

    uint32_t new_ino = alloc_inode();
    uint32_t data_blk = alloc_block();

    Ext2Inode new_disk;
    // ... 清零并初始化 ...
    new_disk.i_mode        = EXT2_S_IFDIR | 0755;
    new_disk.i_size        = block_size_;
    new_disk.i_links_count = 2;  // "." + 父目录的条目
    new_disk.i_blocks      = block_size_ / 512;
    new_disk.i_block[0]    = data_blk;

    write_disk_inode(new_ino, new_disk);

    // 初始化数据块: "." 和 ".."
    auto* dot = reinterpret_cast<Ext2DirEntry*>(dma);
    dot->inode     = new_ino;
    dot->name_len  = 1;
    dot->file_type = static_cast<uint8_t>(Ext2FileType::Directory);
    dot->name[0]   = '.';
    dot->rec_len   = 12;  // 8 hdr + 1 name + 3 pad

    auto* dotdot = reinterpret_cast<Ext2DirEntry*>(dma + 12);
    dotdot->inode     = parent_ino;
    dotdot->name_len  = 2;
    dotdot->file_type = static_cast<uint8_t>(Ext2FileType::Directory);
    dotdot->name[0]   = '.';
    dotdot->name[1]   = '.';
    dotdot->rec_len   = static_cast<uint16_t>(block_size_ - 12);  // 占满剩余空间

    write_block(data_blk);

    add_dir_entry(parent_ino, dir_disk, new_ino, name, name_len, Ext2FileType::Directory);

    dir_disk.i_links_count++;  // 子目录的 ".." 引用父目录
    bgdt_[new_group].bg_used_dirs_count++;
    write_bgdt(new_group);

    write_disk_inode(parent_ino, dir_disk);
    return get_cached_inode(new_ino);
}
```

mkdir 比 create 复杂得多。新目录需要分配一个数据块，写入 "."（指向自身 inode）和 ".."（指向父目录 inode）。"." 的 rec_len 是 12 字节（8 头部 + 1 名称 + 3 对齐填充），".." 的 rec_len 占满块剩余空间——这是 ext2 目录项的经典布局。新目录的 `i_links_count` 初始化为 2（自身 "." 加父目录的条目），同时父目录的 `i_links_count` 要递增（被子目录的 ".." 引用）。BGDT 的 `bg_used_dirs_count` 也要更新。

### unlink() —— 删除目录项并释放资源

```cpp
int Ext2::unlink(uint32_t parent_ino, const char* name, uint32_t name_len) {
    Ext2Inode dir_disk;
    read_disk_inode(parent_ino, dir_disk);

    uint32_t entry_ino = 0;
    remove_dir_entry(parent_ino, dir_disk, name, name_len, entry_ino);

    Ext2Inode target_disk;
    read_disk_inode(entry_ino, target_disk);

    if (target_disk.i_links_count > 0) {
        target_disk.i_links_count--;
    }

    if (target_disk.i_links_count == 0) {
        // 释放直接块 0-11
        for (uint32_t i = 0; i < EXT2_DIRECT_BLOCKS; ++i) {
            if (target_disk.i_block[i] != 0) {
                free_block(target_disk.i_block[i]);
            }
        }

        // 释放间接块及其引用的数据块
        if (target_disk.i_block[EXT2_INDIRECT_BLOCK] != 0) {
            uint32_t indirect_blk = target_disk.i_block[EXT2_INDIRECT_BLOCK];
            if (read_block(indirect_blk)) {
                auto* indirect = reinterpret_cast<uint32_t*>(dma_buf_virt_);
                uint32_t ptrs = block_size_ / sizeof(uint32_t);
                for (uint32_t i = 0; i < ptrs; ++i) {
                    if (indirect[i] != 0) free_block(indirect[i]);
                }
            }
            free_block(indirect_blk);
        }

        target_disk.i_size = 0;
        target_disk.i_blocks = 0;
        write_disk_inode(entry_ino, target_disk);
        free_inode(entry_ino);

        // 如果是目录，更新 bg_used_dirs_count 和父目录 links
        if ((target_disk.i_mode & EXT2_S_IFMT) == EXT2_S_IFDIR) {
            bgdt_[group].bg_used_dirs_count--;
            write_bgdt(group);
            dir_disk.i_links_count--;
        }
    } else {
        write_disk_inode(entry_ino, target_disk);
    }

    // 失效缓存条目
    // 写回父目录 inode
    write_disk_inode(parent_ino, dir_disk);
    return 0;
}
```

unlink 的逻辑是：先从父目录中移除条目，然后递减目标 inode 的链接计数。链接归零时释放所有数据块（直接块 + 间接块引用的块），然后释放 inode 本身。如果是目录，还要递减 `bg_used_dirs_count` 和父目录的 `i_links_count`。最后把目标 inode 的缓存条目标记为失效。

### add_dir_entry —— rec_len 切分

```cpp
bool Ext2::add_dir_entry(uint32_t dir_ino, Ext2Inode& dir_disk, uint32_t entry_ino,
                         const char* name, uint32_t name_len, Ext2FileType file_type) {
    uint32_t required_rec_len = EXT2_DIR_ENTRY_HDR_SIZE + name_len;
    required_rec_len = (required_rec_len + 3) & ~3u;  // 4 字节对齐

    // 扫描现有条目，找有足够空闲空间的条目做切分
    for (uint32_t b = 0; b < total_blocks && b < EXT2_DIRECT_BLOCKS; ++b) {
        // ... 逐条目扫描 ...
        uint32_t entry_min = (EXT2_DIR_ENTRY_HDR_SIZE + entry->name_len + 3) & ~3u;
        uint32_t extra = entry->rec_len - entry_min;

        if (extra >= required_rec_len) {
            // 切分：缩小当前条目，插入新条目
            entry->rec_len = static_cast<uint16_t>(entry_min);

            auto* new_entry = reinterpret_cast<Ext2DirEntry*>(block_data + pos + entry_min);
            new_entry->inode     = entry_ino;
            new_entry->rec_len   = static_cast<uint16_t>(extra);
            new_entry->name_len  = static_cast<uint8_t>(name_len);
            new_entry->file_type = static_cast<uint8_t>(file_type);
            // ... 拷贝名称 ...
            write_block(blk);
            return true;
        }
    }

    // 所有现有块都塞不下 → 分配新块
    uint32_t new_blk = alloc_block();
    auto* new_entry = reinterpret_cast<Ext2DirEntry*>(dma);
    new_entry->rec_len = static_cast<uint16_t>(bs);  // 占满整块
    // ... 填充字段 ...
    write_block(new_blk);
    dir_disk.i_block[new_block_idx] = new_blk;
    dir_disk.i_size += bs;
    write_disk_inode(dir_ino, dir_disk);
    return true;
}
```

ext2 利用 rec_len 的膨胀特性管理空闲空间。目录项的 `rec_len` 可以大于实际最小需要——多出来的空间是"空闲"的。插入新条目时遍历已有条目，找 `extra = rec_len - entry_min >= required_rec_len` 的条目做切分：把当前条目的 rec_len 缩小到最小值，紧随其后创建新条目并继承多余空间。如果所有现有块都塞不下，就分配新块，新条目作为唯一条目占满整块的 rec_len。

### remove_dir_entry —— rec_len 合并

```cpp
bool Ext2::remove_dir_entry(uint32_t /*dir_ino*/, const Ext2Inode& dir_disk, const char* name,
                            uint32_t name_len, uint32_t& out_entry_ino) {
    // ... 逐条目扫描匹配名称 ...

    if (match) {
        out_entry_ino = entry->inode;

        if (pos == 0) {
            // 块首条目：只清 inode（不能合并回）
            entry->inode = 0;
        } else {
            // 合并到前驱条目
            auto* prev = reinterpret_cast<Ext2DirEntry*>(block_data + prev_pos);
            prev->rec_len += entry->rec_len;
        }

        write_block(blk);
        return true;
    }
```

删除比插入简单：找到目标条目后，如果不是块首条目，就把它的 `rec_len` 合并到前驱条目——不需要移动后续数据。如果是块首条目，只能把 `inode` 清零标记为已删除。这是 ext2 目录项设计的优雅之处——用 rec_len 的膨胀/收缩来管理空闲空间，完全避免数据移动。

### alloc_block / alloc_inode —— 位图扫描分配器

```cpp
uint32_t Ext2::alloc_block() {
    for (uint32_t group = 0; group < group_count_; ++group) {
        if (bgdt_[group].bg_free_blocks_count == 0) {
            continue;
        }

        uint32_t bitmap_block = bgdt_[group].bg_block_bitmap;
        read_block(bitmap_block);

        auto* bitmap = reinterpret_cast<uint8_t*>(dma_buf_virt_);

        for (uint32_t byte_idx = 0; byte_idx < bytes_needed; ++byte_idx) {
            if (bitmap[byte_idx] == 0xFF) continue;

            for (uint32_t bit = 0; bit < 8; ++bit) {
                if ((bitmap[byte_idx] & (1U << bit)) == 0) {
                    // 置位、写回位图
                    bitmap[byte_idx] |= static_cast<uint8_t>(1U << bit);
                    write_block(bitmap_block);

                    // 更新超级块和 BGDT 空闲计数
                    --sb_.s_free_blocks_count;
                    --bgdt_[group].bg_free_blocks_count;
                    write_superblock();
                    write_bgdt(group);

                    return global_block;
                }
            }
        }
    }
    return 0;  // 磁盘满
}
```

分配器的工作方式：遍历块组，跳过空闲计数为零的组（快速路径），读该组的位图块，逐字节扫描找第一个 0 bit（空闲）。找到后置位、写回位图、更新超级块和 BGDT 的空闲计数、写回元数据。`alloc_inode` 的逻辑完全对称，只是操作 inode 位图且 inode 号是 1-based。`free_block` 和 `free_inode` 做反向操作：清 bit、递增空闲计数、写回。

### write_superblock / write_bgdt —— 元数据写回

```cpp
bool Ext2::write_superblock() {
    constexpr uint64_t SB_LBA     = EXT2_SUPERBLOCK_OFFSET / EXT2_SECTOR_SIZE;
    constexpr uint16_t SB_SECTORS = EXT2_SUPERBLOCK_SIZE / EXT2_SECTOR_SIZE;

    auto* dma = reinterpret_cast<uint8_t*>(dma_buf_virt_);
    memcpy(dma, &sb_, sizeof(Ext2Superblock));

    if (!ahci_.write(port_index_, SB_LBA, SB_SECTORS, dma_buf_phys_)) {
        return false;
    }
    return true;
}

bool Ext2::write_bgdt(uint32_t group) {
    uint32_t bgdt_start_block  = (block_size_ == 1024) ? 2 : 1;
    uint32_t entries_per_block = block_size_ / sizeof(Ext2BlockGroupDescriptor);
    uint32_t bgdt_block_index  = group / entries_per_block;
    uint32_t entry_in_block    = group % entries_per_block;
    uint32_t disk_block        = bgdt_start_block + bgdt_block_index;

    read_block(disk_block);  // read-modify-write
    auto* block_data = reinterpret_cast<uint8_t*>(dma_buf_virt_);
    memcpy(block_data + entry_in_block * sizeof(Ext2BlockGroupDescriptor),
           &bgdt_[group], sizeof(Ext2BlockGroupDescriptor));
    write_block(disk_block);
    return true;
}
```

超级块写回直接走 AHCI 的扇区级写入（LBA 2, 2 sectors），和 mount 时的读取路径对称。BGDT 写回采用 read-modify-write 策略：先读出包含目标描述符的块，覆盖对应 32 字节，再写回。因为一个块里装了多个描述符（4K 块装 128 个），不能只写一个描述符。

## 设计决策

### InodeOps 子类委托而非直接实现

Ext2DirOps 的 create/mkdir/unlink 只做参数校验后委托给 Ext2 核心方法，不在子类中实现 ext2 逻辑。原因有二：一是 Ext2 的这些方法需要访问私有成员（bgdt_、sb_、inode_cache_），放在 Ext2 类内部更自然；二是路径解析（lookup_in_dir）等内部功能不属于 InodeOps 接口，必须作为 Ext2 的方法存在。把所有 ext2 逻辑集中在 Ext2 类中，InodeOps 子类只做 VFS 适配层。

### 每次分配立即写回元数据

alloc_block 和 alloc_inode 在置位 bitmap 后立即写回超级块和 BGDT。代价是每次分配多 2-3 次磁盘写入，好处是崩溃后元数据一致性有基本保障——不会出现"位图说块已分配但超级块说它空闲"的情况。对教学内核来说这个简单策略够用了，Linux 内核则使用延迟写回和日志来优化性能。

## 扩展方向

- 支持双重间接块（i_block[13]），将最大单文件尺寸提升到约 64MB
- 实现符号链接支持（短 symlink 可存在 inode 的 i_block 区域）
- 优化目录搜索为 HTree 索引，加速大目录查找
- 实现文件系统一致性检查（fsck），检测并修复元数据不一致
- 添加 rmdir 系统调用（当前 unlink 对目录和文件一视同仁）
- 实现文件截断（truncate）：缩短文件时释放尾部数据块
